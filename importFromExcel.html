    • Импорт договоров

    Алгоритм поиска/создания договора 

        //примечание: ОРГ/ФЛ - это ОРГ если в шаблоне указан только ОРГ, это ФЛ если в шаблоне указан только ФЛ,
        это ФЛ если в шаблоне указан и ФЛ и ОРГ

    если (в шаблоне указан ОРГ/ФЛ) {
        010: если (ОРГ/ФЛ найден в системе) {
            ищем договор по "Вид документа+Дата договора+Номер договора+ОРГ/ФЛ"
        } иначе {
            ищем договор по "Вид документа+Дата договора+Номер договора+X"
            (где Х это ФЛ=null если ОРГ указан в шаблоне и не найден в системе,
                 и Х это ОРГ=null если ФЛ указан в шаблоне и не найден в системе)
        }
        020: если на шаге 010 договор найден не был, создаём его
    } иначе {
         ищем существующий договор по "Вид документа+Дата договора+Номер договора",
            если не находим создаём новый
    }

    Импорт коэффициентов аренды

  • алгоритм 
        Х - excel-шаблон "Шаблон реестра договоров"
    А - детализация "Коэф-ты для расчета АП"
    Б - детализация "Объекты договора"
    В - вкладка "Коэф-ты аренды" шаблона Х

    - сначала ищется договор в системе по триаде "ДогНомер"/"ДогДатЗак"/"АрФЛФИО"|"АрОргИмя" (это значения из В)
    - если договор найден {
        - поиск/создание записи (Г) в детализации А:{
            - поиск уже существующей записи на базе триады "ОбИдентиф"/"ОбИдентифЗнач"/"КоэфНазвание"
            если не найдена{
                создание новой
            }
        }
        - наполнение записи Г данными из В, в том числе:{
            - поиск объекта в детализации Б по паре "ОбИдентиф"/"ОбИдентифЗнач"
            если найден{
                заполнение найденным поля "Наименование объекта" детализации А
            }
        }
    }
